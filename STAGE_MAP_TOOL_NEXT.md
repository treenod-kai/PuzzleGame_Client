# 스테이지 맵 툴 다음 설계 메모

기존 골조와 다른 방향으로 다시 만들기 위한 작업 메모다.
내일 작업 시작 시 `MAP.md`, `STAGE_MAP_TOOL.md`, `STAGE_MAP_SCENE.md`를 읽은 뒤 이 문서를 우선 기준으로 삼는다.

---

## 핵심 방향

- 맵 툴은 고정된 `stage_width * stage_height` 그리드를 먼저 만드는 방식이 아니다.
- 메인 맵 공간에 0.5 x 0.5 단위의 클릭 가능한 스팟을 깔고, 스팟 클릭으로 셀을 배치한다.
- 셀은 스팟보다 클 수 있으므로, 새 셀 배치 전 기존 셀과 겹치는지 확인한다.
- 기존 셀이 이미 배치된 영역이 우선이며, 겹치는 새 셀 생성은 막는다.
- 셀 위에 블럭, 생성기, 기믹을 배치한다.
- 셀은 보여지는 셀과 보여지지 않는 셀을 구분할 수 있어야 한다.

---

## 상단 설정 UI

상단은 줄 단위로 구성한다.

### 1줄: 기본 규칙

- 퍼즐 모드 선택
- 클리어 방식 선택
- 클리어 방식은 토글 형태로 선택한다.

### 2줄: 목표 설정

- 타겟 유형 선택
- 타겟 값 선택
- `+` 버튼으로 목표 항목을 가로로 추가한다.
- 목표 항목은 여러 개를 동시에 가질 수 있다.

예시:

```text
[타겟 유형] [타겟] [+] [타겟 유형] [타겟] [+] ...
```

### 3줄: 셀 및 생성기

- 셀 버튼을 누르면 셀 생성 모드가 활성화된다.
- 셀 생성 모드에서 메인 맵 공간의 비어 있는 스팟을 누르면 셀이 생성된다.
- 셀이 이미 있는 영역과 새 셀이 겹치면 생성하지 않는다.
- 생성기 버튼은 셀에 생성기 속성을 부여하는 용도로 둔다.

겹침 예시:

```text
스팟 간격: 0.5 x 0.5
셀 크기: 1 x 1
기존 셀 위치: 1.0, 1.0
클릭 스팟: 1.5, 1.5
결과: 두 셀이 겹치므로 새 셀 생성 안 함
```

### 4줄: 블럭 유형

- 블럭 유형 버튼을 가로 정렬한다.
- 블럭 유형 버튼을 누른 뒤 셀을 누르면 해당 셀에 블럭이 배치된다.
- 블럭 유형 선택 시 기믹 유형 선택은 해제된다.
- 항상 마지막에 누른 유형 버튼만 활성 상태가 된다.

### 5줄: 기믹 유형

- 기믹 유형 버튼을 가로 정렬한다.
- 기믹 유형 버튼을 누른 뒤 셀을 누르면 해당 셀에 기믹이 배치된다.
- 기믹 유형 선택 시 블럭 유형 선택은 해제된다.
- 항상 마지막에 누른 유형 버튼만 활성 상태가 된다.

### 6줄: 기타 설정

- 기타 스테이지 설정을 배치한다.
- 예: 저장, 로드, 테스트, 셀 표시 여부, 스팟 표시 여부, 스테이지 번호 등.

## 메인 맵 공간

- 0.5 x 0.5 단위 스팟을 배열한다.
- 스팟은 클릭 가능한 배치 후보 지점이다.
- 스팟 클릭 동작은 현재 선택된 모드에 따라 달라진다.

동작 우선순위:

1. 셀 생성 모드이면 빈 영역에 셀 생성 시도
2. 블럭 유형 선택 중이면 선택한 셀에 블럭 배치
3. 기믹 유형 선택 중이면 선택한 셀에 기믹 배치
4. 생성기 선택 중이면 선택한 셀에 생성기 속성 배치

셀 생성 조건:

- 클릭한 스팟 기준으로 새 셀의 점유 영역을 계산한다.
- 기존 셀 점유 영역과 겹치면 생성하지 않는다.
- 기존 셀 영역과 겹치지 않으면 새 셀을 생성한다.
- 셀이 있는 위치를 클릭했을 때는 셀 선택 또는 편집으로 처리한다.

---

## 데이터 모델 방향

기존 `StageData` / `CellData` 저장 포맷은 유지한다.
다만 툴 내부 편집 모델은 스팟 기반 배치를 표현할 수 있어야 한다.

필요한 내부 정보:

- 스팟 좌표
- 셀 중심 좌표
- 셀 크기
- 셀 표시 여부
- 셀 타입
- 블럭 ID
- 생성기 블럭 ID 목록
- 기믹 타입 또는 기믹 ID

주의:

- 런타임 저장 시에는 기존 `StageData` / `CellData`로 변환해야 한다.
- 툴 전용 내부 모델을 만들더라도 저장 JSON 포맷을 무단 변경하지 않는다.
- `StageData` 필드만으로 표현이 부족한 기믹/가변 크기 정보는 별도 확장 방식을 먼저 설계한다.

## 현재 골조에서 바꿀 점

기존 구현은 8 x 8 고정 셀을 자동 생성한다.
새 방향에서는 이 방식을 그대로 쓰지 않는다.

수정 대상:

- `StageMapGridView`: 고정 셀 생성 대신 0.5 단위 스팟 표시로 변경
- `StageMapCellView`: 셀 오브젝트 표시 용도로 유지 가능하나 생성 방식 변경
- `StageMapToolState`: 고정 좌표 셀 조회가 아니라 배치된 셀 목록과 점유 영역 조회 중심으로 변경
- `StageMapInputController`: 스팟 클릭과 셀 클릭을 구분하도록 변경
- `StageMapToolController`: 현재 선택 모드가 셀/블럭/기믹/생성기 중 무엇인지 관리
- `StageMapToolPanel`: 1~6줄 UI 구성으로 재작성

삭제 또는 보류:

- 테스트용 키보드 단축키는 정식 UI가 들어오면 제거한다.
- 8 x 8 기본 셀 자동 생성은 새 방식과 맞지 않으므로 보류한다.

## 다음 작업 순서

1. 툴 내부 편집 모델을 먼저 정의한다.
2. 0.5 단위 스팟 그리드 표시를 만든다.
3. 셀 생성 모드와 셀 겹침 판정을 구현한다.
4. 셀 선택과 블럭 배치를 구현한다.
5. 기믹 배치와 생성기 배치를 구현한다.
6. 상단 1~6줄 UI를 구성한다.
7. 내부 모델을 `StageData` 저장 포맷으로 변환한다.
8. 저장/로드/테스트 진입을 다시 연결한다.

## 미정 항목

- 셀 크기 종류: 1 x 1만 쓸지, 여러 크기를 허용할지 결정 필요
- 보여지지 않는 셀의 저장 표현 방식 결정 필요
- 기믹 데이터가 `CellData.panel_id`로 충분한지, 별도 필드가 필요한지 결정 필요
- 클리어 방식과 목표 데이터가 기존 `RuleData.objectives`만으로 충분한지 확인 필요
