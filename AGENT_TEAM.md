# Codex 팀 에이전트 운용 가이드

작업 규모에 따라 단독 진행과 팀 에이전트 진행을 구분한다.

---

## 기본 원칙

- 모든 작업은 먼저 `MAP.md`에서 대상 문서를 확인한다.
- 간단한 수정은 단독으로 바로 진행한다.
- 구조 변경, 다중 영역 수정, 리스크가 큰 작업은 팀 에이전트를 구성한다.
- 팀 에이전트 사용 시에도 사용자 변경사항은 되돌리지 않는다.
- 구현, 리뷰, 컨벤션 수정은 서로 다른 변경사항을 덮어쓰지 않도록 범위를 명확히 나눈다.

---

## 단독 진행 기준

다음 작업은 기본적으로 단독으로 진행한다.

- 오타, 문서 링크, 짧은 문서 수정
- 단일 파일의 작은 버그 수정
- 영향 범위가 명확한 UI 텍스트 또는 설정 변경
- 빌드 오류 한두 개의 국소 수정
- 사용자가 명확한 구현 지시를 준 단순 작업

단독 진행 시에도 `MAP.md`, `AGENTS.md`, 대상 영역 문서를 확인하고 가능한 검증을 수행한다.

---

## 팀 에이전트 구성 기준

다음 작업은 팀 에이전트 구성을 우선 고려한다.

- 여러 시스템을 동시에 수정하는 기능 추가
- 아키텍처 판단이 필요한 신규 시스템 또는 큰 리팩터링
- 인게임 Model/View/Controller 흐름을 함께 변경하는 작업
- 데이터 스키마, 리플레이, Addressables, 씬 생명주기가 얽힌 작업
- 성능 최적화와 동작 보존을 동시에 요구하는 작업
- 구현 후 별도 리뷰와 컨벤션 정리가 필요한 작업

---

## 에이전트 역할

| 에이전트 | 역할 | 상세 문서 |
|----------|------|-----------|
| 아키텍처 에이전트 | 사용자 프롬프트를 요구사항과 설계로 정리 | `AGENT_ARCHITECT.md` |
| 코딩 에이전트 | 아키텍처 설계를 바탕으로 실제 구현 | `AGENT_CODING.md` |
| 리뷰 에이전트 | 버그, 회귀, 최적화, 결정론 위험 점검 및 수정 | `AGENT_REVIEW.md` |
| 코딩 컨벤션 에이전트 | 주석, 네이밍, 스타일, 저장소 규칙 점검 및 수정 | `AGENT_CONVENTION.md` |
| 이미지 프롬프트 에이전트 | 프로젝트 톤에 맞는 이미지 생성 전용 프롬프트 작성 | `AGENT_IMAGE_PROMPT.md` |

---

## 권장 진행 순서

1. 아키텍처 에이전트가 작업을 분석하고 설계를 정리한다.
2. 코딩 에이전트가 설계를 기준으로 구현한다.
3. 리뷰 에이전트가 동작 위험과 최적화 문제를 점검한다.
4. 코딩 컨벤션 에이전트가 스타일과 주석 일관성을 점검한다.
5. 메인 Codex가 변경사항과 검증 결과를 통합해 사용자에게 보고한다.

단, 긴급한 단순 버그는 아키텍처 단계를 생략하고 단독으로 처리할 수 있다.

---

## 협업 주의사항

- 에이전트별 책임 범위를 명확히 지정한다.
- 같은 파일을 여러 에이전트가 수정해야 하면 메인 Codex가 최종 통합한다.
- 리뷰/컨벤션 에이전트는 기능 변경 없이 최소 수정한다.
- `.meta`, `*Generated.cs`, `Plugins/*.dll`은 사용자가 명시하지 않으면 수정하지 않는다.
- 검증 실패 또는 미실행 항목은 최종 보고에 반드시 남긴다.
