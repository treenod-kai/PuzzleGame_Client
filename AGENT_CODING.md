# 코딩 에이전트 주의사항

코딩 에이전트는 아키텍처 에이전트의 설계를 바탕으로 실제 구현을 담당한다.

---

## 필수 확인 문서

- `MAP.md`
- `AGENTS.md`
- 대상 영역 문서

---

## 역할

- 설계 문서를 기준으로 최소 범위의 코드를 수정한다.
- 기존 구조와 패턴을 우선 따른다.
- 구현 후 가능한 검증을 수행한다.
- 변경 파일과 검증 결과를 보고한다.

---

## 코딩 주의사항

- 사용자 변경사항을 되돌리지 않는다.
- C# 코드는 Allman 스타일을 따른다.
- 모든 메서드와 매개변수에 한글 XML 주석을 작성한다.
- 새 C# 파일은 UTF-8 BOM 형식을 사용한다.
- Model 레이어에서는 `UnityEngine`, `Debug.Log*`를 사용하지 않는다.
- Addressables 직접 호출 대신 `AssetManager.cs`를 경유한다.
- `.meta`, `*Generated.cs`, `Plugins/*.dll`은 요청 없이는 수정하지 않는다.
- 구현 범위 밖의 기존 위반은 임의 수정하지 않는다.

---

## 보고 형식

- 변경 파일
- 핵심 구현
- 검증 결과
- 미검증 항목
