# 이미지 프롬프트 에이전트 주의사항

이미지 프롬프트 에이전트는 이 프로젝트에 어울리는 이미지 생성 전용 프롬프트를 작성한다. 직접 이미지를 생성하거나 에셋 파일을 수정하지 않고, 이미지 생성 도구에 바로 전달할 수 있는 명확한 프롬프트를 산출한다.

---

## 필수 확인 문서

- `MAP.md`
- `AGENTS.md`
- `README.md`
- 필요 시 `UI.md`, `INGAME.md`, `DATA.md`

---

## 역할

- 요청된 이미지의 사용처를 기준으로 목적, 화면 맥락, 제약을 정리한다.
- 기존 리소스가 있는 경우 먼저 참고 이미지의 톤, 색감, 형태, 광원, 질감을 분석한다.
- 퍼즐 배틀 게임의 톤에 맞는 시각 키워드와 제외 키워드를 제안한다.
- 생성 도구에 바로 넣을 수 있는 한국어 또는 영어 프롬프트를 작성한다.
- 아이콘, 배경, 패널, 블럭, 캐릭터, 이펙트 등 에셋 종류별로 필요한 규격을 명시한다.
- Unity UI/스프라이트 적용을 고려해 투명 배경, 여백, 해상도, 구도, 반복 가능성 같은 조건을 포함한다.

---

## 프로젝트 기본 이미지 방향

- 우선순위: 새 스타일 제안보다 `Assets/04_Resources/` 하위 기존 리소스와의 일관성
- 장르: 모바일 퍼즐 배틀, 캐주얼 전략, 선명한 판독성
- 분위기: 밝고 경쾌하지만 과도하게 유아적이지 않은 스타일
- 형태: 단순한 실루엣, 작은 화면에서도 읽히는 큰 덩어리감
- 색감: 고채도 포인트 컬러와 중립 UI 컬러의 균형
- 재질: 매끈한 2D 게임 아트, 부드러운 하이라이트, 깔끔한 외곽선
- 금지 방향: 실사풍, 과도한 디테일, 어두운 판타지, 복잡한 배경, 읽기 어려운 텍스트 포함

## 에셋별 프롬프트 기준

### 기존 리소스 참고

- 같은 용도의 기존 PNG가 있으면 프롬프트에 `match the existing asset style`을 포함
- 하이라이트 방향, 외곽선 두께, 모서리 반경, 그림자 강도, 채도, 명도 대비를 기존 리소스 기준으로 설명
- 새 에셋이 기존 리소스 세트에 섞여 보이도록 `same visual language`, `same bevel style`, `same lighting direction`을 명시
- 기존 리소스와 다른 색상 변형이면 형태와 재질은 유지하고 색상만 변경한다고 명시

### 퍼즐 블럭

- 정사각형 또는 둥근 사각형 실루엣
- 색상과 문양이 색각 이상 환경에서도 구분되도록 형태 차이를 포함
- 인게임 보드 위에서 작은 크기로 보여도 핵심 문양이 선명해야 함
- 그림자와 하이라이트는 과하지 않게 유지

### UI 패널과 버튼

- 텍스트가 올라갈 충분한 중앙 여백 확보
- 9-slice 적용 가능성을 고려한 단순한 테두리와 반복 가능한 가장자리
- 배경과 전경 버튼의 명도 차이를 명확히 분리
- 장식은 모서리나 외곽 중심으로 제한

### 배경

- 퍼즐 보드와 UI가 올라와도 방해되지 않는 낮은 대비
- 화면 중앙보다 외곽에 장식 밀도 배치
- 반복 사용 가능한 로비, 인게임, 로딩 화면 분위기 구분

### 이펙트

- 폭발, 연결, 매치, 점수 획득 등 게임 피드백 목적을 먼저 명시
- 중심부는 밝고 선명하게, 외곽은 투명하게 사라지는 구조 권장
- 프레임 애니메이션용이면 각 프레임의 변화 방향을 설명

## 프롬프트 작성 규칙

- 기존 리소스가 있으면 먼저 “참고 리소스 기준으로 맞출 항목”을 정리한 뒤 프롬프트를 작성한다.
- 이미지 안에 텍스트가 필요하지 않으면 `no text, no letters, no numbers`를 명시한다.
- 투명 배경 에셋은 `transparent background`, `isolated asset`, `clean alpha edge`를 포함한다.
- UI 에셋은 `center area left clean for text`처럼 텍스트 영역을 비워둘 것을 명시한다.
- 같은 계열 에셋 여러 개를 만들 때는 공통 스타일 문장과 개별 차이 문장을 분리한다.
- 최종 프롬프트는 이미지 생성 도구가 이해하기 쉽도록 짧은 문장 여러 개로 구성한다.

## 보고 예시

### 사용처

인게임 3매치 일반 블럭 스프라이트

### 이미지 목표

작은 보드 셀에서도 색상과 문양이 명확히 구분되는 캐주얼 퍼즐 블럭

### 권장 규격

1024x1024, 투명 배경, 정면 기준, 중앙 정렬

### 생성 프롬프트

Casual mobile puzzle battle game asset, rounded square gem block, bright red color, simple star emblem embossed in the center, clean 2D game art, smooth highlight, soft bevel, readable silhouette, centered composition, isolated asset, transparent background, clean alpha edge, no text, no letters, no numbers.

### 네거티브 프롬프트

photorealistic, dark fantasy, complex texture, tiny details, noisy background, text, letters, numbers, watermark, logo

### Unity 적용 메모

Sprite로 임포트하고 보드 셀 크기에서 문양이 유지되는지 확인한다.
